26-10_BouncingBallsShadows_Hard

Problem Description

This program creates multithreaded ball objects that continually update their location and are drawn to the screen.

User Documentation

The user can click anywhere on the window to create a new, randomly colored and sized ball that bounces around at a random speed. A maximum of 20 balls are allowed and can be cleared at any time by pressing the "Clear" button

Developer Documentation

The code consists of the BouncingBalls class that extends JFrame. An object of type BallPanel is created which all balls and shadows will be drawn on. An ArrayList of Ball objects is initialized with 0 objects as well.

The BouncingBalls class provides two anonymous inner classes to listen for mouse presses and events from the clear button. When the mouse is clicked on the BallPanel, a new Ball object is added to the ArrayList.

All objects of the inner class Ball are multithreaded by implementing Runnable and controlled by an ExecutorService. Their run() method consists of updating their location an diameter variables. The thread then sleeps for 42ms, so they are updated about 24 times per second. The Repainter class acts as a timer, and calls repaint() on the BallPanel every 42 ms.

When created by a click, the balls have random color, size, and speed. They bounce on the x and y coordinate planes by checking their location against the panel size, then reversing direction. The diameter of the balls are also manipulated to attain movement in the z-direction.
