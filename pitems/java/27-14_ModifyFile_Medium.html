27-14_ModifyFile_Medium

Problem Description

This problem involves both a server-side program that handles requests, and a client-side program to request, edit, and save text files.

User Documentation

If the server-side program is running, the client can be started and used to request a file. If it exists, the file will be shown in the main text area and can be edited and saved back to the server. Additionally, text files can be written directly in the main text area and saved as a specified file name.

Developer Documentation

Text files alice.txt, alice2.txt, and chuck.txt are provided for testing purposes.

The server program creates a ServerSocket object to provide a potential connection for a client to connect to. A Socket object is also created and waits for a connection from a client.

Upon starting, the client program connects to the server and all input and output streams are initialized. The server waits for a command from the client, which can either be a request for a file or a command to save a file.

When given a request for a file, the server creates a File object with the filename provided by the client. It determines if the file exists, and sends either an error message or the full text of the file to the client. The client writes this to its main text area. When asked to save, the server creates a BufferedWriter object along with a File object using the filename from the client, and writes the input from the client to the file.

After the connection to the client is terminated, the server waits for another connection.

When initialized, the client program either connects to the server, or determines there is no server running and closes. When the user specifies a file and presses the open button (or presses enter in the file field), a command to read a file is sent to the client, followed by the file name. The resulting message from the server is read and displayed in the main text area.

When the save button is pressed and a file name is written in the text field, a save command is sent to the server, followed by the file name. Everything in the main text area is then sent to the server to be written to file.
